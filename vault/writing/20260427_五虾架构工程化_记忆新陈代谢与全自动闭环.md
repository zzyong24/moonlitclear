# 五虾架构工程化：记忆新陈代谢与全自动闭环

> 本文记录 2026-04-27 完成的多 Agent 工程化改造，参考月明水清深的架构思路，落地到 OpenClaw 五虾体系。

## 背景

月明水清深分享了一套记忆分层 + 自动新陈代谢的 Agent 架构，核心思路：
- 记忆分三层：指针层 → tag 检索层 → 语义+时间线检索层
- 记忆自动新陈代谢：核心/长期/短期/工作/淘汰记忆，满足条件自动归档
- Skills 自动淘汰+合并机制
- 上下文噪音监控
- Agent 人格系统

本文档记录如何在 OpenClaw 五虾体系里落地这套机制。

## 七大功能模块详解

### T1 - 记忆指针化改造

**目标**：soul.md 只存记忆指针，实际内容归档到 vault/

**架构变化**：
```
SOUL.md (之前)：身份 + 原则 + 记忆内容（混在一起）
SOUL.md (之后)：身份 + 原则 + 指针表（指向 vault/）

vault/ 目录结构：
  vault/events/      → 事件归档
  vault/decisions/   → 决策归档
  vault/people/      → 人物关系归档
  vault/projects/    → 项目归档
  vault/config/      → 配置归档
```

**落地**：6 个 workspace 全部完成指针化改造

---

### T2 - 记忆新陈代谢

**脚本**：`scripts/memory-metabolism.py`

**规则**：
- memory/*.md 文件（30天+）→ 自动移动到 vault/archived/
- vault/archived/*.md 文件（90天+）→ 自动删除
- 核心文件（SOUL.md, MEMORY.md, IDENTITY.md）永不删除

**使用**：
```bash
# 预览模式（默认）
python3 memory-metabolism.py --dry-run

# 实际执行
python3 memory-metabolism.py --execute

# 自定义阈值
python3 memory-metabolism.py --execute --archive-days 7 --delete-days 60
```

**解决的问题**：memory/ 目录无限膨胀，现在自动新陈代谢

### T3 - Skills 淘汰/合并机制

**脚本**：`scripts/skills-lifecycle.py`

**分析结果**：41 个 Skills，286 个文件，全部 90 天内修改过，状态健康

**规则**：
- 使用频率：超过 90 天未修改 → 标记待审查
- 内容重复：相似度 > 80% → 建议合并
- 过时检测：含 deprecated/obsolete 等关键词 → 标记过时

**使用**：
```bash
python3 skills-lifecycle.py --analyze    # 分析并生成报告
python3 skills-lifecycle.py --dry-run   # 预览合并/归档
python3 skills-lifecycle.py --execute   # 执行合并
python3 skills-lifecycle.py --archive    # 归档过时 Skills
```

**解决的问题**：Skills 越来越多没人管，现在有生命周期管理

**文件**：6 个 `PERSONA.md`（每个 workspace 一个）

**六只虾人格差异化设计**：

| 虾 | 哲学底座 | 人格关键词 |

**解决的问题**：每只虾回复风格一致，现在有独特人格

### T5 - 上下文噪音监控

**文件**：`HEARTBEAT.md`（每个 workspace 更新）

**规则**：
```markdown
- 上下文噪音检查：每 30 分钟评估一次
  - 噪音 > 60% → 触发归档建议
  - 噪音 > 80% → 触发立即清理提醒
```

**当前状态**：总管虾上下文噪音约 60%，Growth 旧记录已归档

**解决的问题**：上下文膨胀稀释注意力，现在有量化指标

## 全自动闭环机制

### 架构图

```
cron（每5分钟）
    ↓
cron-auto.py（读取 task-status.json）
    ↓
发现 T? 是 todo + 依赖满足 → sessions_spawn 自动派发
发现 T? 超时 2h+ → 重新提醒
    ↓
subagent 执行 → 结果写回 task-status.json
```

### cron-auto.py 核心逻辑

```python
# 1. 检测 todo 任务（依赖满足时自动触发）
if current_status == "todo" and dependencies_met:
    auto_assign_to_owner()   # 自动派给对应虾
    update_status("in_progress")

# 2. 检测超时的 in_progress 任务
if task.status == "in_progress" and overdue > 2h:
    re_alert_owner()          # 超时重新提醒

# 3. 发现 done → 触发依赖它的下游任务
if task.status == "done":
    cascade_next_task()       # 自动触发依赖它的任务
```

### OpenClaw cron 配置

```json
{
  "id": "agent-engineering-task-check",
  "name": "Agent Engineering 全自动闭环",
  "schedule": { "kind": "cron", "expr": "*/5 * * * *" },
  "payload": { "kind": "agentTurn", "message": "/usr/bin/python3 .../cron-auto.py" }
}
```
