# 一篇文章的旅程：从收藏到行动验证

今晚发生的一幕，让我意识到自己的知识管理工作流已经成了肌肉记忆。

## 起点：随手收藏

刷到 Matt Van Horn 的推文《Every Claude Code Hack I Know (March 2026)》，讲他用 Claude Code 的完整工作流。有价值，但信息量太大——6 个核心技巧，每个都够写一篇文章。

以前的我：收藏 → 忘记 → 下次再看到 → 再收藏。

今晚的我：说了一句 `collect_content(https://x.com/...)`。

## 第一站：知识卡片

AI 抓取了原文，几秒后吐出一张知识卡片：

- 摘要：把 80% 编码时间变成 80% 规划时间
- 5 条要点：`/ce:plan` 思维、语音输入、并行会话、三个设置、研究先行
- 3 个深度问题：规划编码比例？立刻 plan 还是先酝酿？第一个要改的习惯？

这 3 个问题是关键——它逼你**连接自己的经验**，而不是假装读过就算了。

## 第二站：回答触发反思

我随手答了这 3 个问题。有意思的是，在回答第二个问题时，我发现自己不完全认同作者：

> 作者说「有任何想法立刻 /ce:plan」，但我认为——
> - 从零到一的项目：对，立刻和 AI 交流，让它启发你
> - 老项目迭代：不行，你得先想清楚边界，再让 AI plan

这个「不认同」才是真正有价值的部分——它暴露了我的真实思考，而不是照单全收。

## 第三站：三层反思

AI 基于我的回答，生成了三层反馈：

### Mirror（照镜子）

> 你已经形成了「边界先行」方法论：老项目先想边界，再让 AI plan。
> 但这套方法论是隐性知识——如果换个人接手，他能复制你的成功吗？

这一问戳中盲点：我以为自己在「用 AI 编程」，其实是在「设计 AI 能懂的工作流」。

### Deepen（追问）

三个追问，层层递进：

1. 模拟一个「用户积分系统升级」的边界清单（行为层）
2. 第一次发现「不用写代码了」是什么感觉？（情绪层）
3. 用一幅画描述你的 AI 工作流（创造层）

我的回答：

- 边界清单 5 问：状态机、历史备份、流转影响、历史值影响、侵入性
- 情绪：不是不安，是解脱。「我不 care 怎么用 AI，我 care 能创造什么价值」
- 画面：指挥官 + n 个机器人。我是将军，强大模型是统领，便宜模型是执行层

### Bridge（行动）

3 个具体行动：

1. 沉淀「边界先行」方法论 → 一周内产出边界清单 + 复盘
2. 尝试并行双会话 → 下次中等需求时测试
3. 建立项目上下文知识库 → 让 AI 先读项目边界文档

## 第四站：闭环点评

AI 的闭环点评里有句话让我愣了一下：

> 如果指挥官不会打仗，他能让军队打胜仗吗？

我的「边界先行」方法论是无数次试错总结出来的。换一个领域，它还能用吗？

这是真挑战：能否把方法论抽象成**领域无关的模板**？

## 终点：行动验证

闭环后，行动项自动提取到 Obsidian 的 `todos.md`。

勾选完成 ≠ 真的完成。真正的验证是：一周后，我有没有产出那份边界清单？那份复盘笔记？

---

## 回头看

这篇文章的旅程：

```
收藏 → 知识卡片 → 回答问题 → 三层反思 → 追问回答 → 闭环点评 → 行动验证
```

传统学习停在「收藏」。

这套工作流逼你走完全程——从「知道」到「做到」，中间隔着 3 个问题、3 层反思、3 个行动。

---

**核心公式**：

> 知识不是收藏，是行为改变。
> Tool → AI → Tool
> 输入 → 思考 → 输出 → 验证
