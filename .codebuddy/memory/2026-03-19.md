## 修复：Vercel 部署后文章全部消失
- **根因**：vault 目录与 cali.so 是独立的，不在同一个 Git 仓库中。Vercel 只拉 cali.so 仓库代码，vault 文章根本不存在于构建环境
- **方案**：反向 symlink 架构
  - 文章真实文件存放在 `cali.so/vault/writing/`（仓库内部，Git 可追踪）
  - Obsidian vault 中 `writing` 目录是 symlink → `cali.so/vault/writing/`
  - `vault.ts` 路径从 `process.cwd() + '../vault/...'` 改为 `process.cwd() + '/vault/writing'`
  - 清理了 next.config.mjs 中不再需要的 outputFileTracing 配置
- **效果**：Obsidian 写文章 → 自动在 cali.so 仓库中 → git push → Vercel 部署可用
- 原始文件备份在 `vault/space/crafted/writing_backup/`

## 修复：部署后文章仍不显示（第二轮）
- **根因**：虽然文章文件在 Git 仓库中，但 Next.js 的 output file tracing 无法检测 `fs.readFileSync` 动态读取的文件依赖。Vercel serverless function 部署包中不包含 `vault/writing/*.md` 文件
- **方案**：在 `next.config.mjs` 中添加 `experimental.outputFileTracingIncludes`，显式声明 `'/*': ['./vault/writing/**/*']`，强制将文章文件打包进所有 serverless function
- **验证**：本地 build 后检查 `.nft.json` trace 文件，确认 4 篇 md 文件均已包含
- **额外**：在 `vault.ts` 的 `getAllVaultPosts()` 中添加了调试日志（cwd、目录是否存在、文件数量），方便后续排查。部署稳定后可删除
