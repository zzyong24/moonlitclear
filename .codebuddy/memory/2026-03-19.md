## 修复：Vercel 部署后文章全部消失
- **根因**：vault 目录与 cali.so 是独立的，不在同一个 Git 仓库中。Vercel 只拉 cali.so 仓库代码，vault 文章根本不存在于构建环境
- **方案**：反向 symlink 架构
  - 文章真实文件存放在 `cali.so/vault/writing/`（仓库内部，Git 可追踪）
  - Obsidian vault 中 `writing` 目录是 symlink → `cali.so/vault/writing/`
  - `vault.ts` 路径从 `process.cwd() + '../vault/...'` 改为 `process.cwd() + '/vault/writing'`
  - 清理了 next.config.mjs 中不再需要的 outputFileTracing 配置
- **效果**：Obsidian 写文章 → 自动在 cali.so 仓库中 → git push → Vercel 部署可用
- 原始文件备份在 `vault/space/crafted/writing_backup/`
